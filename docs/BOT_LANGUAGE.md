# Shalter Bot Language

`Shalter Bot Language` — встроенный язык сценариев для ботов в `Shalter`.

Он поддерживает два формата:

1. Современный Python-style формат через `ShalterBot(...)`
2. Старый DSL-формат для обратной совместимости

Для новых ботов рекомендуется Python-style синтаксис.

## 1. Быстрый старт

Минимальный рабочий бот:

```python
from shalter import ShalterBot

bot = ShalterBot(
    name="Support Bot",
    niche="Поддержка",
    goal="Помогать пользователю и вести его к следующему шагу.",
    tone="Спокойный, точный, дружелюбный.",
)

bot.greeting("""
Привет! Чем помочь?
""")

bot.rule_contains(["цена", "тариф"], """
Стоимость зависит от задачи. Опиши, что именно нужно.
""")

bot.default("""
Опиши запрос чуть подробнее, и я подберу следующий шаг.
""")
```

Чтобы бот считался валидным, у него должны быть:

- `bot.greeting(...)`
- `bot.default(...)`
- хотя бы одно правило

## 2. Структура программы

Обычно бот состоит из:

1. Импорта:

```python
from shalter import ShalterBot
```

2. Создания объекта:

```python
bot = ShalterBot(...)
```

3. Базовых блоков:

- `bot.greeting(...)`
- `bot.default(...)`
- опционально `bot.guard(...)`
- опционально `bot.handoff(...)`

4. Правил и обработчиков:

- `bot.rule_contains(...)`
- `bot.rule_regex(...)`
- `bot.command(...)`
- `bot.on_command(...)`
- `bot.on_text(...)`
- `bot.on_regex(...)`

## 3. Конструктор `ShalterBot`

Поддерживаемые поля:

```python
bot = ShalterBot(
    name="Имя бота",
    niche="Ниша",
    goal="Главная цель бота",
    tone="Стиль общения",
)
```

Описание полей:

- `name` — имя бота
- `niche` — тематика, ниша или роль
- `goal` — основная задача бота
- `tone` — желаемый стиль ответов

Все поля строковые.

Если часть полей не указана, приложение подставит безопасные значения по умолчанию.

## 4. Базовые методы

### `bot.greeting(text)`

Первое сообщение бота в диалоге.

```python
bot.greeting("""
Привет! Я помогу с тарифами и заявками.
""")
```

### `bot.default(text)`

Ответ по умолчанию, если ни одно правило не сработало.

```python
bot.default("""
Я пока не нашёл подходящий ответ. Опиши задачу подробнее.
""")
```

### `bot.guard(text)`

Защитное замечание, которое добавляется к найденному ответу.

Сейчас оно приклеивается как отдельный блок вида `Важно: ...`.

```python
bot.guard("""
Не обещай цены, сроки и юридические гарантии без подтверждения.
""")
```

### `bot.handoff(text)`

Текст для эскалации на человека.

```python
bot.handoff("""
Если вопрос связан с оплатой, конфликтом или нестандартной ситуацией, передай диалог человеку.
""")
```

## 5. Правила по тексту

### `bot.rule_contains(patterns, reply)`

Срабатывает, если сообщение пользователя содержит один из шаблонов.

```python
bot.rule_contains(["цена", "стоимость", "тариф"], """
Стоимость зависит от задачи. Опиши, что нужно.
""")
```

Аргументы:

- `patterns` — список строк
- `reply` — текст ответа

Особенности:

- сравнение идёт без учёта регистра
- ищется вхождение подстроки
- порядок правил важен

### `bot.on_text(patterns, reply)`

Алиас к текстовому правилу в стиле библиотек для ботов.

```python
bot.on_text(["менеджер", "админ"], """
Если нужен человек, оставь контакт и кратко опиши задачу.
""")
```

Можно передавать:

- список строк
- строковые литералы, из которых будут извлечены шаблоны

## 6. Правила по regex

### `bot.rule_regex(pattern, reply, flags="")`

Срабатывает по регулярному выражению.

```python
bot.rule_regex(r"(ошибка|bug|не работает)", """
Похоже на техническую проблему. Опиши, что сломалось и как это воспроизвести.
""", flags="i")
```

Аргументы:

- `pattern` — строка с regex
- `reply` — текст ответа
- `flags` — строка с флагами, например `"i"`

### `bot.on_regex(pattern, reply, flags="")`

Telegram-style алиас для regex-обработчика.

```python
bot.on_regex(r"^/debug$", """
Команда диагностики принята.
""", flags="i")
```

## 7. Команды

### `bot.command(name, reply)`

Обработчик команды в стиле библиотек для Telegram-ботов.

```python
bot.command("start", """
Привет! Это команда /start.
""")
```

Поддерживается:

- `start`
- `/start`

То есть команда срабатывает и с косой чертой, и без неё.

### `bot.on_command(name, reply)`

Алиас для `bot.command(...)`.

```python
bot.on_command("help", """
Напиши вопрос, и я подскажу, куда идти дальше.
""")
```

## 8. Строки

Язык понимает два вида строк:

### Короткая строка

```python
bot.greeting("Привет! Чем помочь?")
```

### Многострочная строка

```python
bot.default("""
Опиши задачу подробнее.
Можно указать:
- цель
- проблему
- ожидаемый результат
""")
```

Также поддерживаются raw-строки для regex:

```python
bot.rule_regex(r"^/start$", "Принято", flags="i")
```

## 9. Порядок выполнения

При сообщении пользователя бот работает так:

1. Берёт программу
2. Компилирует её
3. Проверяет правила сверху вниз
4. Возвращает ответ первого совпавшего правила
5. Если задан `guard`, добавляет его к ответу
6. Если совпадений нет, возвращает `default`

Это значит:

- порядок правил очень важен
- узкие правила лучше ставить выше общих

Пример:

```python
bot.rule_contains(["оплата"], """
Подскажу по оплате.
""")

bot.rule_contains(["оплата картой"], """
Подскажу по оплате картой.
""")
```

В таком порядке сработает первое правило. Более точное правило лучше поднять выше.

## 10. Ошибки компиляции

Редактор показывает ошибки, если:

- нет `bot.greeting(...)`
- нет `bot.default(...)`
- нет ни одного правила
- у `bot.rule_contains(...)` пустой список шаблонов
- у `bot.on_text(...)` нет шаблонов
- у `bot.rule_regex(...)` нет regex-шаблона
- у `bot.on_regex(...)` нет regex-шаблона
- у `bot.command(...)` пустое имя команды
- у правила пустой ответ
- regex синтаксически неверен

## 11. Что реально поддерживается сейчас

Текущий рантайм понимает:

- `from shalter import ShalterBot`
- `bot = ShalterBot(...)`
- `bot.greeting(...)`
- `bot.default(...)`
- `bot.guard(...)`
- `bot.handoff(...)`
- `bot.rule_contains(...)`
- `bot.rule_regex(...)`
- `bot.command(...)`
- `bot.on_command(...)`
- `bot.on_text(...)`
- `bot.on_regex(...)`

Сейчас не поддерживаются как отдельные конструкции языка:

- переменные
- `if/else`
- циклы
- пользовательские функции
- inline-кнопки
- callback query
- состояния диалога
- память по шагам

Это можно расширить дальше отдельными обновлениями.

## 12. Полный пример

```python
from shalter import ShalterBot

bot = ShalterBot(
    name="Sales Support Bot",
    niche="Продажи и поддержка",
    goal="Понимать запрос, давать первый ответ и вести пользователя к следующему шагу.",
    tone="Спокойный, точный, вежливый.",
)

bot.greeting("""
Привет! Я бот внутри Shalter. Напиши, что тебе нужно, и я постараюсь помочь.
""")

bot.command("start", """
Привет! Это стартовая команда. Можешь спросить про тарифы, заявку или ошибку.
""")

bot.on_command("help", """
Я умею отвечать по ключевым словам, командам и regex-правилам.
""")

bot.guard("""
Не придумывай неподтверждённые цены, сроки и юридические обещания.
""")

bot.rule_contains(["цена", "стоимость", "тариф"], """
Стоимость зависит от задачи. Опиши, что именно нужно, и я помогу сориентироваться.
""")

bot.on_text(["менеджер", "человек", "оператор"], """
Если нужен человек, оставь контакт и коротко опиши задачу. Я подготовлю передачу диалога.
""")

bot.rule_regex(r"(ошибка|bug|не работает|crash)", """
Похоже на техническую проблему. Опиши:
1. Что сломалось
2. Что ожидалось
3. Какие шаги к этому привели
4. Есть ли текст ошибки
""", flags="i")

bot.on_regex(r"^/debug$", """
Команда диагностики принята.
""", flags="i")

bot.handoff("""
Если вопрос касается оплаты, конфликта, безопасности или нестандартной ситуации, передай его человеку.
""")

bot.default("""
Опиши задачу чуть подробнее, и я подберу для тебя следующий шаг.
""")
```

## 13. Старый DSL

Для старых ботов всё ещё поддерживается старый формат:

```txt
name "Bot"
niche "Support"
goal "Помогать пользователю"
tone "Спокойный"

greeting """
Привет!
"""

rule includes "цена" "тариф" """
Подскажу по стоимости.
"""

rule regex "/(bug|ошибка)/i" """
Похоже на баг.
"""

default """
Уточни вопрос подробнее.
"""
```

Поддерживаемые старые блоки:

- `name`
- `niche`
- `goal`
- `tone`
- `greeting`
- `default`
- `guard`
- `handoff`
- `rule includes`
- `rule regex`

Для всех новых ботов лучше использовать Python-style формат.

## 14. Рекомендации

- держи `greeting` коротким и понятным
- ставь самые точные правила выше
- используй `guard`, если бот не должен обещать лишнее
- `default` делай спокойным и направляющим
- для команд используй `bot.command(...)`
- для FAQ чаще подходит `bot.rule_contains(...)`
- для техподдержки чаще подходит `bot.rule_regex(...)`
